# Separate members from auth users

CSK Choir Hub will model `Member` separately from the auth `User`, while v1 still requires every Member to be backed by exactly one User through a unique `userId`. We intentionally keep choir-domain relations out of the autogenerated auth `User` model; with Prisma 7.8 this means the v1 `Member.userId` link is represented as a required unique scalar rather than a Prisma relation, preserving auth schema ownership while giving member lifecycle, group membership, positions, attendance, and future profile fields a stable home.
