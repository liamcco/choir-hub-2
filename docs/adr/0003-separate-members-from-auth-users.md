# Separate members from auth users

> Superseded by [ADR-0013](./0013-unify-users-and-choir-identity.md).

CSK Choir Hub will model `Member` separately from the auth `User`, while v1 still requires every Member to be backed by exactly one User through a unique `userId`. We intentionally keep choir-domain relations out of the autogenerated auth `User` model; preserving auth schema ownership while giving member lifecycle, group membership, positions, attendance, and future profile fields a stable home.
