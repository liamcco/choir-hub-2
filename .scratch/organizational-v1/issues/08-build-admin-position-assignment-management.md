# 08 — Build admin Position Assignment management

**What to build:** Let admins assign Positions to Members over dated periods, end assignments, view current and historical holders, and enforce one holder at a time per Position through the write interface.

**Blocked by:** 04 — Enable admin account and Member management; 07 — Build admin Position and Position Scope management.

**Status:** ready-for-agent

- [ ] Admins can assign a Position to a Member with a start date.
- [ ] Admins can end a Position Assignment with an end date.
- [ ] Admins can view the current holder of a Position.
- [ ] Admins can view historical holders of a Position.
- [ ] Admins can view a Member's current and historical Position Assignments.
- [ ] Overlapping assignments for the same Position are rejected through the write interface.
- [ ] Position Assignment does not require extra metadata such as appointment source, notes, or handover date in v1.
- [ ] The UI prioritizes existing shadcn/ui-style components and is responsive.
