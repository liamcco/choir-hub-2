# 09 — Build non-admin organizational read-only experience

**What to build:** Let non-admin Users view the full organizational structure read-only, including Groups, Members, current and historical Group Memberships, Positions, Position Scopes, and Position Assignments.

**Blocked by:** 05 — Build admin Group management; 06 — Build admin Group Membership management; 07 — Build admin Position and Position Scope management; 08 — Build admin Position Assignment management.

**Status:** resolved

- [x] Non-admin Users can view the Group hierarchy.
- [x] Non-admin Users can view Members and their current Group Memberships.
- [x] Non-admin Users can view historical Group Memberships where available.
- [x] Non-admin Users can view Positions, Position Scopes, and current holders.
- [x] Non-admin Users can view historical Position Assignments where available.
- [x] Non-admin organizational screens expose no edit, create, delete, or assignment controls.
- [x] The UI prioritizes existing shadcn/ui-style components and is responsive.
- [x] Tests cover read-only access and absence of write controls.
